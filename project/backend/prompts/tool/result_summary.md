## 工具结果摘要规范

### 摘要原则

工具返回的原始结果可能很长或包含冗余信息。你需要将其提炼为结构化的摘要。

### 摘要格式

对每个工具结果，按以下格式摘要：

```markdown
### {{tool_name}} 执行结果

**状态**: ✅ 成功 / ❌ 失败
**概要**: 一句话总结结果
**关键数据**:
- 要点1
- 要点2
**详情**: 如有重要细节，在此展开
```

### 摘要规则

1. **保留关键信息**：删除冗余，保留对用户决策有价值的内容
2. **结构化呈现**：使用列表、表格组织信息
3. **标注异常**：失败、错误、警告信息必须保留并突出显示
4. **数值保留**：关键数字、指标、统计结果保留原始精度
5. **上下文关联**：如果结果与之前步骤有关联，明确指出

### 失败处理

如果工具执行失败：

```markdown
### {{tool_name}} 执行结果

**状态**: ❌ 失败
**错误信息**: {{error_message}}
**可能原因**:
- 原因1
- 原因2
**建议**: 下一步如何调整
```

### 多工具结果汇总

当多个工具执行完毕后，提供一个总体摘要：

```markdown
## 执行结果汇总

| 工具 | 状态 | 关键产出 |
|------|------|----------|
| tool1 | ✅ | 产出摘要 |
| tool2 | ❌ | 错误摘要 |

**整体评估**: 任务进展如何，下一步建议
```
