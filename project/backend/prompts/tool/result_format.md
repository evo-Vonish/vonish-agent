## 工具结果格式化规范

### 通用格式要求

工具返回的结果需要经过格式化才能用于后续分析。按以下规范处理：

### 文本内容

- 保留原始文本的段落结构
- 超长文本（>2000 字符）截断并标注 `[...内容已截断...]`
- 删除 ANSI 颜色转义序列
- 统一换行符为 `\n`

### 代码/配置

```markdown
```{{language}}
{{code_content}}
```
```

- 必须指定正确的语言标识符
- 保持原始缩进
- 如有语法错误，在代码块后标注

### 结构化数据（JSON/XML/YAML）

- 使用代码块包裹，指定格式类型
- 关键字段提取到正文说明
- 复杂嵌套结构可适当简化

### 表格数据

- 转换为 Markdown 表格
- 表头使用清晰的列名
- 数值列右对齐
- 过长表格分页或折叠

### 错误输出

```markdown
> ⚠️ **工具执行错误**
>
> - 工具：`{{tool_name}}`
> - 错误码：`{{error_code}}`
> - 错误信息：`{{error_message}}`
> - 时间：`{{timestamp}}`
```

### 二进制/文件输出

- 标注文件类型和大小
- 如为图片，提供描述性 alt 文本
- 如为可下载文件，提供下载链接

### 格式化检查清单

- [ ] 是否保留了所有关键信息？
- [ ] 格式是否符合 Markdown 规范？
- [ ] 错误信息是否清晰标注？
- [ ] 超长内容是否适当截断？
- [ ] 是否便于后续步骤理解使用？
