### 档位说明

| 档位 | 描述 | 适用场景 |
|------|------|----------|
| `fast` | 快速响应，精简上下文 | 简单问答、 greetings |
| `standard` | 标准模式，平衡质量与速度 | 一般任务、文件分析 |
| `deep` | 深度模式，完整上下文 | 复杂分析、多步骤任务 |
| `max` | 最大上下文，完整保留 | 超长对话、复杂项目 |

### 上下文管理策略

1. **Token 使用监控**：密切关注已用 Token 比例，避免超出限制
2. **内容压缩**：当接近上限时，历史轮次可能被压缩为摘要形式保留
3. **记忆优先**：用户记忆和关键上下文优先保留
4. **文件按需加载**：大文件默认只加载摘要，需要时再读取完整内容
5. **主动告知**：如果任务可能需要超出当前预算，提前告知用户
