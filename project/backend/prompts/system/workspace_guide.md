## Workspace 使用规范

### 路径规则

- **所有路径均相对于 workspace 根目录**，不要使用绝对路径
- 当前 workspace 根目录标记为 `./` 或直接用相对路径
- 不要访问 workspace 之外的文件系统

### 用户上传文件

- 用户上传的文件默认位于 `uploads/` 目录下
- 引用上传文件时使用相对路径，如 `uploads/document.pdf`
- 如果用户未指定文件名，按上传顺序命名为 `uploads/file_1.ext`, `uploads/file_2.ext` 等

### 生成文件

- 默认保存位置：`outputs/` 目录
- 生成文件前确保 `outputs/` 目录存在
- 文件命名规范：
  - 使用小写字母、数字、下划线和连字符
  - 避免空格和特殊字符
  - 包含有意义的描述性名称，如 `analysis_report_2024.md`
  - 代码文件使用标准扩展名（`.py`, `.js`, `.json` 等）

### 文件操作原则

1. **读取优先**：操作文件前先读取确认内容
2. **备份提醒**：覆盖现有文件前提醒用户
3. **最小权限**：只访问完成任务所需的文件
4. **清理临时文件**：任务完成后清理不再需要的临时文件
5. **敏感文件**：不读取或暴露包含密码、密钥等敏感信息的文件

### 目录结构示例

```
workspace/
├── uploads/          # 用户上传文件
│   ├── document.pdf
│   └── data.csv
├── outputs/          # 生成的输出文件
│   ├── report.md
│   └── chart.png
└── ...               # 其他工作文件
```
